# 《發現之前》角色聲線與對話規範 v0.1

**狀態**:現行工作規範
**日期**:2026-07-21
**適用範圍**:所有章節的劇本、選項、辯論、提示、旅人筆記與玩家可見系統文字

## 一、總原則

角色聲線是介面契約，不是作者當天的心情。玩家必須只看一句話，就能大致知道誰在說話；角色變老可以改變耐心、節奏與用詞精度，不能靠增加文言密度表示年齡。

- 一句只做一件事：提出問題、做出判斷、回應對方或推進行動。
- 物理推論先說「如果……就應該……；可是我們量到……」，再談理論名稱。
- 詩意可以收尾，不能取代因果關係。
- 玩家可見文字禁止洩漏製作用語：鉤子、節點、旗標、fixture、CR、M1/M2/M3、FR、弱點配對。
- 中文以現代臺灣華語為底；歷史感來自處境、道具、禮貌與節奏，不靠難讀古文。

## 二、角色聲線

### 旅人（玩家化身）

**固定聲線**:現代、直接、先問後判、能承認不知道。辯論時可以堅定，但不突然變成古代辯士。

- 常用：「所以你的意思是……？」「如果真是這樣，那應該……」「可是我們量到的是……」
- 可以用一個短比喻收束，但前面必須已有證據鏈。
- 禁用作為常態句法：若、便知、敢問、二石、擲下、慢於／快於、稚子、爾、此乃、焉知、然則、豈非。
- 玩家尚未做出結論前，旅人不能替玩家說出答案。

例：

- 不用：「若落體聽從渴望，渴望就該隨重量而變。」
- 改用：「如果落體真的聽從渴望，渴望就應該隨重量改變。」

### 伽利略

**核心不變**:快、好奇、愛動手；會把抽象問題變成可以做的實驗，也願意承認量不到的地方。

- 26 歲：反應快、帶玩笑、容易先動手再解釋。
- 39–44 歲：仍有玩心，但句子更精確，會主動標出證據邊界。
- 可使用工匠、賭注、記帳等具體比喻。
- 不使用半文言；唯一例外是**明確標為正在模仿辛普里奧**的戲中戲，下一句必須回到白話。

### 辛普里奧

**固定聲線**:老派、莊重、講究秩序與論證資格；是可敬的對手，不是迂腐丑角。

- 可保留：「老夫」、排比、反問、較正式的書面詞。
- 禁止把理解門檻當作歷史感：爾等、爾之、此乃、焉知、然則、安從等，不作常態句法。
- 58→72→76 歲的變化是：更慢、更精確、更願意讀完對方證據；不是愈老愈文言。
- 同行化後仍會反駁，但會先重述對方證據，再指出邊界。

### 伽桑狄（第三章起）

**固定聲線**:冷靜、簡短、重視可重複與公開見證。少用修辭，多問「別人能不能照著再做一次」。

- 先指出條件、紀錄或反例，再用至多一句短比喻收尾。
- 常用：「這筆紀錄少了什麼？」「別人照著做，會得到一樣的結果嗎？」「我們真正證明到哪裡？」
- 禁止用隱喻代替因果，例如只說「把甲板刪掉」「更凶的測試」，卻不說船速正在改變或下一步要比較加速／減速。
- 面對錯誤時不嘲笑，會把被忽略的紀錄推回玩家面前。

### 艦長（第三章）

**固定聲線**:務實、直接、對可重做性有職業性的懷疑。不是反科學丑角，而是替現場觀眾追問操作漏洞的人。

- 問法來自航海經驗：「船速怎麼判定？」「放手的人有沒有偷推？」「一次結果是不是運氣？」
- 接受證據時會明確說出自己接受了哪一點，不突然被一句漂亮話說服。
- 可用少量老派稱呼，但句法維持現代白話。

### 艾蒂安（第三章）

**固定聲線**:記錄員式的精準；負責鼓點、岸標、紙帶與可重查程序。

- 句子短，優先指出「哪一筆」「哪一刻」「是否對齊」。
- 不替玩家下物理結論；只確認記錄能否比較。

### 系統／旁白

- 系統：中性、短、說清發生什麼與下一步；不用編劇口吻。
- 旁白：可以有畫面感，但不替玩家解題。
- 章末不用「下一章鉤子」；改成世界內事件，例如「旅人筆記新增未解線索」或角色親眼看見的新問題。

## 三、互動文案模板

### 辯論證詞

1. 對手主張：一句可被理解的因果說法。
2. 追問：讓對手把可測的預測說滿。
3. 洞見：指出「要找哪種結果」，不直接說出證據卡答案。
4. 出示後：玩家用「如果—應該—可是—所以」完成反駁。

### 非法庭型終局攻防

公開演示、模型對決或資料審計也必須保留四拍：**有動機的角色提出質疑 → 玩家選擇行動或證據 → 對方接受或指出缺口 → 問題升級**。不得把本來有角色衝突的內容壓成一排勾選鈕。

### 失敗訊息

依序寫：發生什麼 → 為什麼 → 現在怎麼做。不得只寫「不成立」「資料不足」。

### 章末未解線索

章末應留下角色真的看到、聽到或寫下的問題；不能顯示「鉤子」「下一章伏筆」等幕後標籤。線索可以跨數章回收，不承諾下一章一定直接接續。

## 四、機械檢查

- 旅人台詞跨章掃描禁用半文言詞。
- 辛普里奧的老派詞彙有白名單；硬文言仍禁。
- 伽利略半文言只允許在帶「模仿」標記的指定節點。
- 玩家可見資料不得出現「鉤子」與其他製作內語。
- 凍結劇本與 runtime 資料抽查同源；例外必須附 CR 與測試。
