## 1. 交付與環境(R-ENV)

- **R-ENV-01**:交付為靜態單頁應用;解壓後以現代桌面瀏覽器(Chromium/Firefox 現行版)**直接雙擊開啟 `index.html` 即可遊玩**,無建置步驟、無網路請求。
- **R-ENV-02**:附自動化測試,單一指令可執行(`npm test`,無外部依賴;或開啟 `tests/tests.html`),涵蓋:fixture 載入、判定規則、狀態轉移、矩陣一致性(R-DATA-03)、鏡像一致性(R-DATA-05)。瀏覽器層驗收依零依賴紀律以人工清單簽核(`docs/browser-checklist.md`)。
- **R-ENV-03**:repo 結構含 `/src` `/data` `/tests` `/docs` 與 `decisions.md`(GB-ADR)。
- **R-ENV-04**:介面為灰盒保真度:系統字體、原生控件、純文字;禁止裝飾性美術。
- **R-ENV-05**:因 file:// 之 opaque origin 限制,執行期**不得以 fetch/XHR 載入資料**;資料檔以 `/data/*.js`(全域賦值,傳統 script 標籤)供頁面載入,並同置 `/data/*.json` 規範鏡像。兩者一致性由 R-DATA-05 保證。

## 2. 遊戲流程總覽

單頁含三個面板,玩家可自由切換:**實驗台**、**旅人筆記**、**辯論廳**。目標:在實驗台產出數據 → 在筆記中親自指認規律並建立主張(E3)→ 在辯論廳擊破支柱 P3。反向亦允許:未備證據即進辯論,依 R-DEB-08 產生生產性失敗。

## 3. 實驗系統(R-EXP)

### 3.1 配置空間

| 維度 | 選項 |
|------|------|
| 球 | 銅大/銅小/木球 |
| 槽面 | 打磨/原木 |
| 傾角 | 緩/中/陡 |
| 計時 | 水鐘/脈搏/音格 |

- **R-EXP-01**:玩家自由組合 3×2×3×3 任一配置並執行;每次執行使「天數計數器」依計時工具遞增:**脈搏 +1、水鐘 +2、音格 +3**(器材整備成本)。天數僅累計顯示,v0 不設上限、不觸發事件;結束畫面揭示總天數(R-UI-02)。此為**可比較的路徑成本軸**:總天數=Σ(每次執行之工具成本),不同過關路徑天數結構相異,供玩家於結束畫面比較;是否升格為策略軸(上限/事件/取捨目標),俟形成性試玩證據再議。
- **R-EXP-02**:每次執行回傳四段等時距增量讀值 Δ1–Δ4(單位:刻度,顯示至 0.1),自動存入筆記為 run 紀錄(含完整配置、讀值、序號、當時天數)。
- **R-EXP-03**:讀值由確定性組合模型(§4)產生;同一 session 內,同配置第 n 次執行得該配置第 ((n−1) mod 3)+1 號預設樣式。新開 session 從 1 號起算。

### 3.2 通過矩陣(規範)

「可判定性」依 §5 判定規則;**銅大/銅小 同判**,下表「銅」二者皆適用(打磨槽面):

| 配置 | 單次 | 三次平均 |
|---|---|---|
| 緩+水鐘 / 緩+音格 / 中+水鐘 / 中+音格 / 陡+音格 | 通過 | 通過 |
| 緩+脈搏 | **不通過** | **通過**(廉價工具×多次) |
| 陡+水鐘 / 陡+脈搏 / 中+脈搏 | 不通過 | 不通過 |
| 任何「原木」槽面 | 不通過 | 不通過(摩擦型特徵) |
| 木球(任意其他配置) | 不通過 | 不通過(離群型特徵) |

- **R-EXP-04**:通過矩陣為**規範**;附錄 A 數值為基準初值。衝突時以矩陣為準,修數值並記 changelog。
- **R-EXP-05**:至少三條可通過路徑且成本結構相異——**水鐘路徑**(緩/中皆可單次判定,2 天/次)、**音格路徑**(全傾角單次可判,3 天/次,最貴)、**脈搏路徑**(僅緩傾角,三次平均,1 天/次,最廉)。實作不得使任一路退化為不可行。
- **R-EXP-06**:三類失敗配置之可分辨特徵——
  **摩擦型**(原木):讀值系統性偏低、後段流失比例遞增,且三個 run 形狀互不一致;
  **計時型・水鐘**:首段(最短時距)讀值系統性偏高+小幅抖動,隨傾角嚴重度放大,**平均不可消除**;
  **計時型・脈搏**:大幅抖動+輕度系統偏移,**平均可大幅消除**(廉價工具×多次路徑之機制基礎);
  **離群型**(木球):每 run 一個游走位置的突刺。
  特徵由附錄 A 樣式實現;不得以無結構亂數取代。
  **代理聲明**:計時誤差樣式(含水鐘首段偏高)為本灰盒**用來表現短時測量不確定性的教學代理**,其設計目的是產生「愈快愈難測、可分辨、水鐘型平均不可除/脈搏型平均可除」的失敗特徵;**不宣稱為伽利略水鐘實測誤差之重建**(其歷史程序為全程/指定距離集水稱重,與本切片之逐段增量讀值不同)。此差異依憲章四於史實頁揭露(史實頁不在本切片,義務已入 `greybox/docs/debt.md`)。

## 4. 誤差組合模型(R-DATA)

  - B:緩 (12,36,60,84)/中 (14,42,70,98)/陡 (22,66,110,154)。
  - S 嚴重度:水鐘 緩0.5・中1・陡2.5;脈搏 緩1・中2.5・陡5;音格 全0.5。
  - 各維度樣式為 3 組×4 段之固定偏移表,數值見附錄 A(與 v0.1.1 相同,經兩輪審稿驗證)。

- **R-DATA-03**:內建「矩陣一致性測試」:**54 種配置全枚舉**(3球×2槽×3傾角×3計時),各配置之單次(3 run 逐一)與三次平均判定結果須與 §3.2 矩陣完全相符。

## 5. 規律指認與主張系統(R-JUD)

原則不變:系統做記帳,玩家做看見;系統**永不**主動宣告規律。

- **R-JUD-01**:玩家自筆記勾選 1–3 筆 run 組成「判定選集」。選集之**配置指紋為四維:球、槽面、傾角、計時工具**;任一維相異即顯示相異變因清單並拒絕受理,附語:「變因未受控,無法認證」(教學回饋,非懲罰)。重複或無效之 run ID 一律拒絕受理。
- **R-JUD-02**:選集可切換「累計/增量」視圖,多筆 run 可疊示同一表格;**視圖切換不得改變玩家已勾選之選集**。工具僅呈現數字,不標示規律。另設比較工具:選 2 筆 run 顯示其配置相異維度(存入 comparisons)。
- **R-JUD-03**:玩家輸入「第五段增量預測值」。以選集平均 Δ̄1 為基準,|預測 − 9·Δ̄1| ≤ 12%·9·Δ̄1 即預測命中(含入邊界);同時選集須通過內部一致性:Δ̄i/Δ̄1 與 (3,5,7) 之偏差各 ≤12%。兩者皆過 → 產生主張紀錄(claim),標注配置指紋與當時天數。
- **R-JUD-04**:指認失敗僅回覆「預測未中/選集內部不一致」與偏差幅度,不解釋原因、不給正解。
- **R-JUD-05**:斷言操作——
  - **E3.a 規律成立**:任一主張成立即點亮。
  - **E3.b 與球重無關**:選兩筆成立之主張執行斷言;系統驗證兩者**僅「球」相異(銅大/銅小)且槽面、傾角、計時三維全同**→ 點亮。不符時列出實際相異變因。
  - **E3.c 隨傾角形式不變**(選做):同上,僅「傾角」相異、其餘三維全同 → 點亮。
  - 斷言型別僅接受 b/c;其他輸入回覆錯誤,不改變任何狀態。
- **R-JUD-06**:E3 狀態 = {a,b,c};「E3 確立」= a∧b。

## 6. 論辯系統(R-DEB)——支柱 P3 單場

### 6.1 資料

辛普里奧支柱 P3 三句證詞(佔位文,語意保留):s1 重物性喜就下/s2 渴望愈強行愈疾,十斤之鐵思鄉十倍/s3 目的之因,數字豈能言之。弱點:**s2 × E3.b**。證物欄固定:E3(展開子項)、E1 存根、S1 存根。

### 6.2 規則

- **R-DEB-01**:說服力量表初始 5。
- **R-DEB-02**:追問免費,揭示補充佔位文;s2 追問文含「重量」語意線索。
- **R-DEB-03**:出示 = (證據,子項,目標證詞);E3 必須展開選子項。
- **R-DEB-04**:(E3, b, s2) 且 E3.b 已點亮 → s2 破裂 → P3 破裂 → 勝利畫面。
- **R-DEB-05**:(E3, a, s2) 且 b 未點亮 → 量表不減;證詞追加需求佔位文(「單一球重之規律,何證與重量無關?」)。
- **R-DEB-06**:其餘任何三元組 → 量表 −1,附吸收佔位文。
- **R-DEB-07**:量表歸零 → 辯論中止;顯示缺口提示;證據與主張完整保留。
- **R-DEB-08**:E3 未確立可入場;失敗自己教學。

## 7. 三層狀態機(R-STA)

- **R-STA-01** `evidenceState`:runs[](配置、讀值、序號、天數)、E3{a,b,c}、存根清單。
- **R-STA-02** `inferenceState`:claims[](選集、預測值、判定結果、四維配置指紋、天數)、assertions[]、comparisons[]。
- **R-STA-03** `beliefState`:simplicio.P3{s1,s2,s3 各 intact|broken;persuasion 0–5;status pending|broken|suspended}。
- **R-STA-04 關鍵轉移表**:

| 事件 | 守衛 | 狀態變更 |
|---|---|---|
| 執行實驗 | — | runs 追加;天數 += 工具成本(1/2/3) |
| 判定 | 選集四維指紋一致 | claims 追加(成敗皆記) |
| 斷言 b | 兩 claim 僅球相異(餘三維全同)且皆成立 | E3.b=true |
| 出示正解 | E3.b=true | s2=broken;P3=broken |
| 出示不足 | E3.a=true ∧ E3.b=false | 無量表變更;s2 需求文旗標 |
| 錯誤出示 | — | persuasion−1;=0 時 status=suspended |
| 再入辯論 | status=suspended | persuasion 重置 5;證詞狀態保留 |

- **R-STA-05**:狀態存於記憶體;所有轉移為純函式可測。
