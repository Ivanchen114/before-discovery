## 一、章節驗收句

玩家必須親手完成以下推理，NPC 不得提前代答：

1. 若沒有向內作用，月球會沿當下切線離開，不會自行繞圓。
2. 軌道可由原有前進與每一刻指向中心的偏折共同形成；重力持續改變速度向量，在近圓模型中主要表現為方向改變。
3. 月球約在六十個地球半徑外；若作用按距離平方縮弱，地表一秒落下量與月球六十秒偏離切線的量落在同一量級。
4. 距離律不能只配合一筆月球資料；鎖定後還要預測未揭露的火星、木星週期。
5. 同一反平方模型能以較少額外補丁處理月球、行星與高傾角彗星，勝過本章明列的簡單共轉渦旋版本。
6. 模型的定量成功支持一套可計算規則，仍不能證明引力的作用機制，也不能抹去 Hooke、Halley、Flamsteed 的貢獻。

刪掉玩家操作後，K1–K5 任一證據不得自動成立。

### R-GRAV4-01 確定性

- 引擎內部採固定步長與固定初始條件，不使用 `Math.random()`。
- 同一初始條件、偏折向量與距離律必須得到逐點相同的軌跡、週期與殘差。
- UI 不得自行改座標；所有座標由引擎回傳。
- 教學用整數尺度與現代驗算值分層保存，不可在介面中偷換：

### R-GRAV4-03 偏折向量容忍帶

偏折操作不是像素描準：

- 方向：向量與指向中心方向夾角 ≤ `15°`。
- 量級：相對該步模型建議量的 `0.75–1.25`。
- 滑鼠／觸控：拖曳箭頭端點。
- 鍵盤：方向鍵調方向，`[`／`]` 或畫面按鈕調量級，Enter 提交。
- 觸控熱區不得小於 44 CSS px。

若超出容忍帶，引擎仍接受該向量並演出結果；不得提交後立刻彈「錯誤」遮住軌跡。

### R-GRAV4-04 距離律與週期

內部模型定義作用強度 `a(r) ∝ 1/r^n`。近圓軌道的週期比由引擎以

```text
T(r) / T(1) = r^((n + 1) / 2)
```

計算，但玩家介面不要求代入公式。玩家只看到：

- 天體跑完一圈所需的模型時鐘；
- 與觀測時鐘逐步錯開的角度；
- 數值殘差與允許帶。

通過門檻：

- 月球偏折相對誤差 ≤ `5%`。
- 火星週期相對誤差 ≤ `3%`。
- 木星週期相對誤差 ≤ `3%`。
- 三者皆通過才可提出 K2、K3。

UI exponent 刻度可連續調整 `0.0–3.0`，顯示至小數一位；通過帶預期約落在 `1.9–2.1`，不得只提供「不變／反比／反平方」三按鈕。

## 四、軌道工作台 R-ORB4

### R-ORB4-01 畫面構成

1. 中央地球圓盤。
2. 月球／試體。
3. 已走軌跡實線。
4. 當下切線預測虛線。
5. 玩家偏折箭頭。
6. 速度箭頭與「下一拍若不改向」幽靈點。
7. 時間鼓點與重試。

所有軌跡、箭頭、位置與殘差由 HTML Canvas 或等效程式圖層即時計算；不得用生成圖片寫死答案。

### R-ORB4-02 可見後果

- `tangent-escape`：至少續跑四拍，直到距地心增加到初始值 `1.8×`。
- `outward-escape`：保留切線幽靈，讓玩家看見實際軌跡比無偏折更遠。
- `impact`：續跑到軌跡穿入地球邊界，留下撞點。
- `unstable`：完成至少一個明顯內外擺動後才停。
- `closed`：至少顯示一圈；不宣稱數值積分得到完美圓。

錯誤提示須指向可見量（方向、箭長、殘差），不得只說「再想想」。

### R-ORB4-03 自由重試

- 成功前後皆可「用同一初始條件重畫」。
- 每次嘗試保留縮圖紀錄，最多顯示最近四筆；引擎歷程可全留。
- 重試不回到劇情、不扣資源、不增加虛構天數。
- 讀檔恢復在工作台，不重播已確認的時間轉場。

## 五、跨尺度工作台 R-SCALE4

### R-SCALE4-01 操作順序

1. 顯示地表一秒落體紙帶。
2. 玩家把視野拉到 `60 R⊕`，月球卡出現。
3. 玩家把比較時間從 1 秒拉到 60 秒。
4. 系統顯示月球觀測偏折帶，但不說應選何種距離律。
5. 玩家調距離律並執行；每次都畫出預測偏折。
6. 至少比較兩筆不同律後才可鎖定。

距離與時間拉桿必須真的改變紙帶比例、預測位置與標尺，不得只換一句說明。

### R-SCALE4-02 單位誠實

- 教學主畫面可用「地球半徑」「秒」「公尺」。
- 史實頁說明當代量測與現代標準化資料的差異。
- `4.9 m` 是教學近似，不顯示成無誤差真值。
- 不把地表重力 `g` 在整段 60 秒都視為常數來模擬真實穿地落體；此處比較的是月球距離處的局部短時間偏折。

### R-SCALE4-03 鎖律與未揭露資料

- 鎖律前火星、木星只顯示名稱與距離，不顯示週期。
- 鎖律後按「讓模型先跑」才存預測；其後才翻觀測卡。
- 解鎖重試會保留舊預測為劃線紀錄，防止看到答案後假裝沒選過。
- K3 只認鎖定在揭露前的預測。

## 六、模型比較 R-MOD4

### R-MOD4-01 比較對象

`inverseSquare`：

- 以同一距離律與各天體初始位置／速度計算路徑。
- 允許更換觀測初始條件，不允許每個天體另造距離律。

`simpleVortex`：

- 明列為「同心、共轉、隨距離分層的簡單渦旋版本」。
- 可帶行近圓、低傾角行星。
- 若要配合高傾角彗星，必須新增局部流層或穿透例外；每項新增假設在畫面留下補丁標記。

介面與台詞必須持續稱「這個簡單渦旋版本」，不得把比較結果擴張成所有笛卡兒式模型。

### R-MOD4-02 三案例

| 案例 | 兩模型共同任務 | 分岔 |
|---|---|---|
| Moon | 維持環繞地球 | 兩者均可定性成功，不作決勝 |
| Planets | 同時配合 Earth／Mars／Jupiter 週期 | 反平方同律通過；簡單渦旋需調多層速度 |
| Comet 1680 | 讓同一物體以高傾角穿越行星區域並連成入／出路徑 | 反平方沿同律計算；簡單共轉渦旋需額外穿透或改流補丁 |

### R-MOD4-03 K4 斷言

正確主張：

> 在這三組資料與明列假設下，反平方模型用同一條距離律留下較小殘差與較少額外補丁，因此具有較強的統一與預測力。

拒絕的過強主張：

- 「渦旋已被邏輯證明不可能。」
- 「有較少補丁的模型必然是宇宙最後真相。」
- 「能畫出橢圓就已解釋引力機制。」

## 七、印刷台組合證明 R-PRF4

### R-PRF4-01 五個槽

| 槽 | 必需來源 | 畫面作用 |
|---|---|---|
| 原有運動 | M2／M3 帶入或 fallback 紙條 | 建立切線箭頭 |
| 向內改向 | K1 | 讓軌道開始彎 |
| 距離律 | K2 | 決定不同距離的箭長 |
| 未揭露預測 | K3 | 接上 Mars／Jupiter 時鐘 |
| 跨模型反驗 | K4 | 接上 comet 與殘差頁 |

放錯位置時，對應的幾何圖必須真的斷開、倒轉或失去尺度；不得只把卡片染紅。

未玩前章時：

- fallback 紙條以 15 秒內可讀的最短摘要提供。
- 明示「這是前案摘要，不是你親手量得的紀錄」。
- 科學判定相同，特殊角色台詞不同。

### R-PRF4-02 信用分配

必須完成四項映射：

| 貢獻 | 人物 |
|---|---|
| 切線運動＋指向中心吸引的書信提問 | Hooke |
| 1684 追問、催稿與出版推動 | Halley |
| 行星／彗星觀測 | Flamsteed |
| 數學證明與跨天體整合 | Newton |

旅人不在信用名單；玩家的功績保存為「重做並驗證證明鏈」。

### R-PRF4-03 最後邊界

可選主張至少三項：

1. `mechanismSolved`：「我們已證明引力如何穿過空間作用。」→ 可見反例：證明鏈所有頁都只記運動與定量關係，機制槽仍空白；拒絕並可重答。
2. `newtonAlone`：「Newton 一人完成全部概念、觀測與出版。」→ 信用連線斷裂；拒絕並可重答。
3. `ruleEstablished`：「我們建立了一套跨地表與天空、可計算且可被新資料反驗的規則；作用機制仍未決。」→ 取得 K5。

失敗不扣分，但 `overclaimTried` 留在旅人筆記，讓完章回述可反思。

## 八、敘事資料與 lint R-NAR4

### R-NAR4-01 資料同源

- 正式 runtime 後，台詞唯一來源為 `scenes4.json`。
- 劇本 Markdown 由同一資料核對，不允許 runtime 另寫一套不同歷史因果。
- 場景資料每行至少含 `sceneId`、`speaker`、`text`、`kind`、`historyTag`。
- `historyTag`：`documented`、`reconstruction`、`fictional-bridge`、`later-recollection`、`modern-model`。

### R-NAR4-02 禁止句

- 「一顆蘋果讓 Newton 發現萬有引力。」
- 「Hooke 只有猜想，沒有任何真正貢獻。」
- 「Halley 只是替 Newton 出書。」
- 「Flamsteed 只是提供數字。」
- 「反平方律解釋了引力為什麼存在。」
- 「所有渦旋模型都已被這顆彗星證偽。」
- 「月球做的是等速圓周運動。」（若未加近似條件）
- 「向心力把月球拉回原位。」（應說持續改變速度方向）

### R-NAR4-03 延後術語

K1 前不要求玩家使用「向心加速度」。可見語言先用：

- 原本要走的切線；
- 每一拍往中心改多少；
- 離原切線多遠；
- 距離變遠時，作用縮弱得多快。

K1 後在筆記命名「向心偏折／向心加速度」；K2 後才命名「反平方」。

## 九、存檔與章節路由 R-SAV4

- 獨立存檔鍵：`before-discovery:chapter4:v1`。
- 章節完成鍵沿用系列 rail，不覆寫 ch1–ch3。

## 十二、最低測試門檻

### 12.1 引擎

1. 同輸入逐點同軌跡，無 RNG。
2. 無偏折必定切線逃逸。

5. 三拍合格偏折後續跑形成閉合帶。
6. 未完成後果演出前不得顯示答案提示。
7. 任一失敗可重試，且重設同一初始條件。
8. 完成軌跡不自動取得 K1。
9. K1 必須同時引用切線與閉合軌跡。
10. 距離／時間拉桿會改變數值與座標，不只改文字。
11. `n=0`、`n=1`、`n=2` 在 60/60 fixture 分別產生約 `17640 m`、`294 m`、`4.9 m`。

## 十四、待審查項

1. 離散積分方法、步長與閉合帶門檻須由物理／狀態機雙查確認，避免數值誤差被誤教成自然現象。
2. 1680 comet 與簡單渦旋的比較需史實審查確認台詞不超過當時資料與 Newton 實際論證。
3. 火星／木星標準化 ratio 應在正式 fixture 附來源與有效位數，不假裝是 Flamsteed 單人原始表格。
4. 1665 場景是必要情緒入口，還是會放大蘋果神話，送總監裁決；本稿推薦保留但明確拆神話。
5. K2、K3 是否應合成一張證據，需依試玩節奏判斷；本稿先分開，因「配合已知資料」與「預測未揭露資料」認知責任不同。
