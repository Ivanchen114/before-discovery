## 3. B 級

### B-1｜動態選取項與狀態更新對讀屏使用者不可辨識

**位置**：`src/ui.js:33-57, 96-100`；`index.html:30-40, 62, 75, 81, 90, 117`。

run／claim checkbox 由 `innerHTML` 生成，只有 `class` 與 `data-id`，沒有 label 或 `aria-label`；讀屏只會遇到一串未命名勾選框。`labMsg`、`compareMsg`、`judgeMsg`、`assertMsg`、狀態列與辯論紀錄更新後也沒有 `role=status`、`role=log` 或 `aria-live`，關鍵成功／失敗不會自動被宣告。頁籤只靠 `.active` 視覺樣式，沒有 `aria-selected`／`aria-pressed` 或面板關聯。

**最小修補**：給每個 checkbox 以 run／claim 編號和配置組成的可存取名稱；結果區使用克制的 `aria-live="polite"`／`role="status"`，辯論紀錄用 `role="log"`；頁籤補可讀狀態與 `aria-controls`。不需要改視覺設計。

### B-2｜切換增量／累計視圖會無提示清除已勾選的 run

**位置**：`src/ui.js:33-45, 121-123`。

**重現路徑（由 DOM 轉移可直接驗證）**：執行至少兩筆 run → 在筆記勾選兩筆 → 切換「增量／累計」radio → `renderRuns()` 清空 `tbody.innerHTML` 並重建所有 checkbox → 原選取消失 → 按比較或認證會被告知重新選取。視圖轉換不應改變判定選集，否則玩家會把表現層操作理解成資料狀態改變。

**最小修補**：重繪前保存已選 ID 並在新列恢復 checked，或只更新四個數值欄而不重建選取欄。補一項 DOM／人工驗收測試。

### B-3｜`assertE3` 將任何非 `b` 型別默認為 `c`

**位置**：`src/engine.js:137-158`。

**重現**：建立兩筆僅傾角不同且成立的 claim，呼叫 `assertE3(state, "x", [1,2])`；實測回傳 `assertion.ok=true` 且 `e3.c=true`。原因是 `type === "b" ? "球" : "傾角"` 將所有其他輸入歸入 c 分支。UI 目前只送 b/c，所以不是現行玩家阻擋；但這與「純函式引擎是可信狀態邊界」的維護目標不符。

**最小修補**：函式入口只接受 `b`／`c`，其他型別回傳不改 state 的錯誤；新增 invalid-type 測試。

### B-4｜R-ID 覆蓋仍偏向成功代表例，邊界與 UI 驗收未鎖定

`tests/suite.js` 已覆蓋核心，但以下高價值邊界尚未成為常駐測試：

- R-JUD-01：0 筆、4 筆、重複 ID、不存在 ID；
- R-JUD-03：預測與一致性**恰為 12%**時的包含邊界；
- R-DEB-06：「其餘任何三元組」的全組合表，而非數個代表例；
- R-DEB-07：中止／再入前後 `runs`、`claims`、assertions 的深度相等性，目前只明確比對證詞；
- R-UI-01／02 與 R-ENV-01：三面板同步、勝利兩題、總天數、雙擊開啟及鍵盤完整通關。

**建議**：優先補引擎邊界與三元組表測試；UI 以一份不依賴網路的瀏覽器驗收清單起步。這些不是現有 16 項失敗，而是防後續章節回歸的缺口。

### B-5｜第四種計時工具仍需跨四檔手動同步，資料驅動只完成一半

**位置**：`data/patterns.js/.json`、`index.html:54-59`、`tests/suite.js:17-28`。

新增工具必須更新雙載體、HTML option 與測試列舉／期望矩陣。JS／JSON 雙載體是 GB-ADR-001 為離線交付作的合理取捨，且已有漂移測試，不建議推翻；真正可減少的是 HTML 的硬編碼。

**替代方案與成本**：在資料載體加入可顯示的 timer metadata，由 UI 產生 `<option>`；測試的期望工具集合仍獨立維護，避免「實作漏項、測試也跟著漏」的同源盲點。一次性改動小，新增工具時可少改 `index.html`；仍保留 `.json` 鏡像與一致性測試。

## 4. C 級

### C-1｜全量重繪在本切片可接受，長局需觀察

每次 `renderAll()` 都重建 runs、claims 與 statements。30–40 分鐘切片若累積大量 run，會造成焦點位置與選取狀態更難維持；目前資料量小，先記為 C。B-2 修正時可順便保留焦點與選取。

### C-2｜灰盒視覺紀律與數字可讀性通過

介面使用系統字體、原生控制項、無裝飾性美術；表格數字右對齊並啟用 tabular numerals，符合灰盒判讀需求。active 頁籤另有粗體＋底線，不單靠顏色；鍵盤焦點也沒有被 CSS 移除。
