## 2. A 級問題

### A-1｜R-MODE-02 把 E3.c 限於學者，R-DEB-12 卻要求探索玩家用 E3.c 前提過關

**位置**：R-HOOK-02、R-DEB-12、R-MODE-02。

**可驗證反例**：

1. 以探索模式完成 M2，合法終態為 E3.a=true、E3.b=true、E3.c=false。
2. 進入 R-DEB-12 的探索兩步，第一步卻要回答「各傾角皆成立／增陡不改形式」。
3. 這不是玩家已取得的證據；規格也沒有任何系統事件在探索模式補發 E3.c。

**因果**：終辯的核心證據鏈存在未定義前提，違反憲章二與三。若 M3 照字面實作，只能由 NPC 洩題或讓玩家假裝做過。

**最小修補**：E3.c 改為兩模式必經；探索模式提供引導式傾角複驗，學者模式保留自由配置與三槽論證。R-HOOK-02、R-MODE-02、R-DEB-12 及驗收測試同步改。

### A-2｜R-NAR-05 與前期錯誤理論不可相容；GB-ADR-006 的例外又過寬

**位置**：R-NAR-05；GB-ADR-006；`tests/narrative-suite.js` 的 lint。

**可驗證反例**：劇本 A1-1 必須讓伽利略先陳述自己錯誤的密度理論，其中含「同材質落得一樣快」。R-NAR-05 卻規定所有 NPC 禁詞白名單節點都必須位於玩家指認之後，所以合法史實立場無法通過原規格。

M2 以 `class=pre-discovery/post-reveal` 解衝突，方向合理；但現行測試只對字面等於 `post-reveal` 的條目檢查 `afterPlayerNode`。把 class 寫成任何其他字串都能跳過拓撲守衛，白名單會從「受控例外」退化成自由通行證。

**因果**：規格若不收回此偏差，測試全綠不能證明刪台詞紀律；新增章節可用錯拼或第三種 class 靜默洩題。

**最小修補**：正式採納語義分類並寫入 v0.1.1：

- enum 限定為 `character-hypothesis`／`post-reveal`；未知 class 測試必敗。
- `character-hypothesis` 必填 `reason` 與 `refutedBy`，且 `refutedBy` 指向可達的玩家驗證／反駁節點；不得用於 system／stage 的正典結論。
- `post-reveal` 必填並驗證 `afterPlayerNode`。
- lint 明載為防回歸工具，不取代人工語義審稿。

## 3. B／C 級問題

### B-1｜R-TWR 的場地選擇沒有可感知差異

R-TWR-01 要玩家選鐘樓／城牆，R-TWR-03 卻只有固定塔上演出，R-TWR-04 只定義見證人差異。這違反規範 2 防偽條款。應讓 `site` 至少改變場景文本、等待成本或見證可信度；否則刪除 q2。

### B-2｜R-REP-02 同時要求「顯示鎖定選項」與「強制進修復」，外部行為不唯一

目前 M2 是信譽歸零後立即轉入 SC-R1，因此玩家看不到「關鍵 NPC 選項被鎖」的畫面。請二選一寫清：

- 立即導流：顯示一次鎖定原因 toast／系統行後進 SC-R1；或
- 留在原頁：選項顯示鎖定，玩家確認後進 SC-R1。

建議採前者，流程較短；並把 R-NAR-03「不滿足即隱藏」對鎖定提示的例外寫明。

### B-3｜R-SAV-02 要求提示，M2 實作目前靜默重開

壞檔備份策略應保留，方便除錯且成本極低；但 parse／schema 失敗後須在標題頁顯示一次「舊進度無法讀取，已備份並開新局」，不能只讓「繼續」消失。提示應可測，不要用 modal 阻塞。

### B-4｜R-END-02 的史實標籤集合不足以承載劇本資料

規格只列史實／傳說／戲劇改編／教學代理；劇本另有「合理重建」「史實記載但解釋有爭議」「考據性重建、非定論」。請定義可擴充 enum 或將它們正規化，避免 M3 把不確定性壓扁成「史實」。

### C-1｜法源版本需在凍結前更新

現稿仍引用劇本 v0.2 與 ADR-005／006／007。凍結版應指向修後劇本 v0.2.2，並補 ADR-008（評注本）與 ADR-009（P-5／美術解凍覆寫）；功能範圍仍可維持零美術，不必把美術規格塞進本文件。

## 4. 上游問題清單裁定

### 4.1 辯論再入從第一根未破支柱續

**採納。** 已破支柱不重打，才能讓失敗保留資訊並符合憲章五；再入時以一行回顧已破支柱即可。說服力重置 5、證據與支柱狀態保留。

### 4.2 死路 B 診斷兩模式共有

**採納。** 診斷是理解數據品質的核心因果，不是可從探索模式拿掉的提示福利。模式差異應落在配置自由度與論證組裝，而非剝奪必要回述。

### 4.3 壞檔備份

**採納備份，不同意靜默處理。** 保留 `_corrupt` 鍵並加非阻塞提示，見 B-3。

### 4.4 GB-ADR-006 白名單分類

**部分採納。** 必須分類「角色假說」與「玩家揭曉後覆述」，但 class 要封閉列舉、錯字必敗，角色假說還需可驗證的後續反駁節點，見 A-2。
