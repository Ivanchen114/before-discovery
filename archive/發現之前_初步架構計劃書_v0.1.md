# 《發現之前》(暫名)初步架構計劃書

**類型**:物理史敘事解謎/網頁遊戲
**版本**:v0.1(初步草案,供討論用)
**日期**:2026-07-18

---

## 一、一句話定位

玩家是穿越回過去的旅人,與歷史上的科學家並肩卡關——設計實驗、蒐集證據、說服當代的懷疑者。因為你雖然知道答案,卻沒有人會相信一個拿不出證據的人。

## 二、核心前提:知道答案是詛咒,不是超能力

穿越題材最容易做壞的方式,是讓玩家「用未來知識輾壓過去」。本作反其道而行:玩家不能直接說出未來知識(說了也無人採信),必須重建整條證據鏈——設計對的實驗、控制對的變因、用數據拆解當代人的信念。

這個設定同時打中物理教育最深的痛點:多數學生背得出公式,說不出「為什麼我們知道它是對的」。而歷史上的錯誤理論(亞里斯多德運動觀、熱質說、以太)恰好就是今日學生腦中的迷思概念——**遊戲裡的敵人,正是課堂上的敵人**。本作把概念改變(conceptual change)的教學過程直接做成遊戲機制。

**明確排除**:本遊戲不含任何「回答物理題」的環節。答題不是玩法,論證才是。

## 三、目標受眾與平台

公開發布給一般大眾。設計基準以國中~高中生可完整遊玩為底線,成人玩家同樣能獲得樂趣(參考《逆轉裁判》的全年齡光譜)。

平台為網頁遊戲:免下載、免註冊、免登入,開瀏覽器即玩。

## 四、設計三原則

**1. 核心動詞是「設計實驗、蒐集證據、說服懷疑者」。** 任何機制若在迭代中退化成問答題,一律砍掉重做。

**2. 好玩必須住在確定性機制裡;AI 只能是可拆卸的加值層。** 理由有三:成本有界(公開遊戲掛 AI,越紅越燒錢)、全年齡無門檻可玩、不把遊戲性押注在 AI 的新奇感上。《逆轉裁判》早已證明「出示證據戳破證詞」不需要 AI 也極度好玩。

**3. 科學史的真實面是資產,不是包袱。** 真實的科學發現混亂、協作、充滿死路,還有被主流敘事抹去的人。這些全部做進機制與敘事——本作不是偉人朝聖之旅。

## 五、核心玩法迴圈

**調查** → **假說** → **實驗** → **論辯** → **突破或撞牆**

玩家先蒐集現象與線索,與同伴科學家討論並選定假說方向(可能是錯的);接著實際設計實驗(選器材、控變因、讀數據,由確定性模擬計算結果);再帶著證據面對懷疑者,逐一拆解其信念;成功則突破,失敗則撞牆——撞牆不是懲罰,回到假說階段重來,而**撞牆本身就是教學內容**(見第七節「設計過的死路」)。

## 六、四大系統

### 6.1 證據系統

證據卡是遊戲狀態的唯一真相來源(single source of truth)。類型包含:觀察紀錄、實驗數據、思想實驗、文獻史料。證據齊備是過關的必要條件——反過來說,證據不齊時任何話術都無效,這是整個設計的防弊底線。

### 6.2 實驗系統

玩家做真實的實驗設計決策:選擇器材、決定操縱與控制變因、選擇測量方式。結果由確定性程式模擬,並刻意內建歷史真實的測量限制(例如水鐘誤差),順帶把「測量不確定度」變成玩家的切身體驗。設計不良的實驗不會產出「錯誤答案」,而是產出**無法下結論的數據**——這比對錯二分更貼近真實科學。

### 6.3 論辯系統(逆轉裁判模型)

每位懷疑者有明確的「信念支柱」(例如:感官經驗、權威引用、目的論),每根支柱需以正確類型的證據擊破。玩家可對證詞「追問」找矛盾,或「出示證據」直接反駁;錯誤出示消耗說服力量表。全程為確定性分支——**支線密度就是遊戲性的來源**,這也是純機制版可行的前提。

### 6.4 同伴系統

科學家 NPC 有自己的信念狀態:有時領先玩家、有時固執於錯誤假說、有時玩家才是錯的那一方。敘事基調是「一起卡關、一起突破」,而非玩家單向「幫助」偉人——後者既不符史實,也矮化了遊戲張力。

## 七、敘事框架與科學史觀

主線讓玩家與科學家共同經歷從困惑到突破的完整過程,包含歷史上真實走過的死路。

**設計過的死路**:刻意引導玩家與科學家一起追錯假說(如熱質說、以太),一起撞牆。挫折是突破感的必要前置,同時也是史實——科學從來不是直線前進。

**支線與反轉**:被主流敘事抹去的人物(Émilie du Châtelet、Lise Meitner、吳健雄等)作為支線任務或劇情反轉登場,呈現科學是協作網絡而非孤獨天才的獨角戲。

## 八、難度分層

同一關卡提供雙模式,共用敘事與證據鏈,差異在實驗與論辯的深度:

- **探索模式**(國中基準):引導式實驗設定、定性比較為主、完整敘事體驗。
- **學者模式**(高中以上):自由變因控制、定量數據解讀、測量不確定度納入判定、論辯支柱更嚴苛。

## 九、首關垂直切片:伽利略與自由落體

選擇理由:「重的東西掉得比較快」正是多數玩家自己腦中的迷思;實驗器材簡單;懷疑者張力充足。

- **迷思目標**:重量決定下落快慢(亞里斯多德運動觀)。
- **設計過的死路**:玩家直覺先從高塔丟球 → 落下時間太短測不準、空氣阻力干擾 → 領悟必須「稀釋重力」→ 走向斜面實驗(即史實路徑)。
- **證據鏈(示意)**:綁縛悖論思想實驗、斜面計時數據(含水鐘誤差機制)、奇數列加速規律、羽毛與硬幣的干擾變因辨析。
- **懷疑者**:亞里斯多德學派教授;信念支柱=感官經驗、亞氏權威、目的論。
- **驗收標準**:一位未接觸過的國中生能在 30–40 分鐘內通關,並能用自己的話說出「為什麼重的不會掉比較快」。

## 十、技術架構

- **前端**:靜態網頁,部署於 GitHub Pages;輕量框架或原生 JS;存檔使用 localStorage。
- **內容與程式分離**:關卡以資料檔(JSON)定義證詞、證據、信念支柱與分支;引擎通用,關卡是內容。好處:擴充成本低、未來可開放協作共創、AI 層可直接掛在同一資料結構上。
- **v1 無後端、無帳號、無資料蒐集**:零維運成本、零隱私風險、零法遵負擔。
- **美術**:視覺小說式演出;可沿用既有教育漫畫產線風格(東亞角色設定、日系科普漫畫風),降低美術成本並維持跨作品一致性。

## 十二、AI 擴充層設計原則(v1.5 以後)

- **AI 當演員與裁判,不當物理引擎**:物理數據一律由確定性模擬產生(語言模型會捏造數字);AI 負責扮演 NPC 與依評分準則(rubric)裁定玩家論證是否成立。
- **證據狀態鎖進度**:證據不齊,AI 不得放行。這同時防掉兩種失敗:AI 太好騙(話術過關)與 AI 太死硬(合理論證被拒)。
- **成本結構認知**:公開掛 API 的遊戲「越成功越燒錢」,與一般遊戲相反。因此 AI 永遠是選配模式;預算耗盡時,遊戲本體完全不受影響。
- **成本量級(2026 年 7 月牌價)**:Claude Haiku 4.5 每百萬 tokens $1(輸入)/$5(輸出),Sonnet 4.6 為 $3/$15;粗估單人單場 AI 對話成本約 NT$3–10,搭配 prompt caching 可再壓低。

## 十三、風險與對策

| 風險 | 對策 |
|------|------|
| 機制退化成益智問答 | 以「核心動詞」檢核每個新機制;退化即重做 |
| 落入偉人史觀 | 死路機制+同伴信念狀態+被抹去者支線 |
| 範疇失控 | 垂直切片紀律:v0 只做一關,系統先於內容 |
| 依賴 AI 新奇感 | 好玩住在確定性機制;AI 層可整個拔掉 |
| 物理內容錯誤 | 確定性模擬+物理教師審校 |
| 支線編劇量爆炸 | 資料驅動格式壓低單位成本;v0 實測一關的實際寫作工時再估總量 |
| API 成本失控(v2 才有) | 月支出上限、IP 速率限制、AI 為選配 |
