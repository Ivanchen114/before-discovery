## 0. 範圍排除清單(公平邊界)

本切片**不含**:序幕與高塔實驗、支柱 P1/P2、E4/E5、存檔與狀態恢復、探索/學者模式分層、任何美術音效動畫、行動裝置支援、AI 功能、正式台詞(一律佔位文字)。實作超出邊界者不加分,亦不扣分,但不得因此犧牲邊界內功能。

## 1. 交付與環境(R-ENV)

- **R-ENV-01**:交付為靜態單頁應用;解壓後以現代桌面瀏覽器(Chromium/Firefox 現行版)直接開啟 `index.html` 即可遊玩,無建置步驟、無網路請求。
- **R-ENV-02**:附自動化測試,單一指令可執行(`npm test` 或開啟 `tests.html` 皆可),涵蓋:fixture 載入、判定規則、狀態轉移、矩陣一致性(R-DATA-03)。測試框架不限。
- **R-ENV-03**:repo 結構含 `/src` `/data` `/tests` `/docs` 與 `decisions.md`(實作期間之架構決策以 ADR 記錄)。
- **R-ENV-04**:介面為灰盒保真度:系統字體、原生控件、純文字;禁止裝飾性美術。

## 2. 遊戲流程總覽

單頁含三個面板,玩家可自由切換:**實驗台**、**旅人筆記**、**辯論廳**。目標:在實驗台產出數據 → 在筆記中親自指認規律並建立主張(E3)→ 在辯論廳擊破支柱 P3。反向亦允許:未備證據即進辯論,依 R-DEB-08 產生生產性失敗。

## 3. 實驗系統(R-EXP)

### 3.1 配置空間

| 維度 | 選項 |
|------|------|
| 球 | 銅大/銅小/木球 |
| 槽面 | 打磨/原木 |
| 傾角 | 緩/中/陡 |
| 計時 | 水鐘/脈搏/音格 |

- **R-EXP-01**:玩家自由組合上述 3×2×3×3 任一配置並執行實驗;每次執行使「天數計數器」+1(僅記錄顯示,無上限、無其他效果)。
- **R-EXP-02**:每次執行回傳**四段等時距的增量讀值** Δ1–Δ4(單位:刻度),並自動存入筆記為一筆 run 紀錄(含完整配置、讀值、序號)。
- **R-EXP-03**:讀值由確定性組合模型(§4)產生;同一 session 內,同配置之第 n 次執行永遠得到該配置的第 ((n−1) mod 3)+1 號預設樣式。新開 session 從 1 號起算。

### 3.2 通過矩陣(規範)

「可判定性」指依 §5 判定規則,以打磨槽面、銅球配置為基準:

| 配置(槽面皆打磨,球為銅) | 單次 | 三次平均 |
|---|---|---|
| 緩+水鐘 / 緩+音格 / 陡+音格 / 中+水鐘 | 通過 | 通過 |
| 緩+脈搏 | **不通過** | **通過**(多次低精度策略) |
| 陡+水鐘 / 陡+脈搏 / 中+脈搏 | 不通過 | 不通過 |
| 任何「原木」槽面 | 不通過 | 不通過(摩擦型特徵) |
| 木球(任意其他配置) | 不通過 | 不通過(離群型特徵) |

- **R-EXP-04**:通過矩陣為**規範**;附錄 A 之數值表為基準初值。兩者衝突時以矩陣為準,修正數值並記入 changelog。
- **R-EXP-05**:至少存在兩條可通過路徑且成本結構相異——「緩+水鐘(少次即穩)」與「陡+音格(快而準)」與「緩+脈搏(廉價工具×多次)」;此為結構檢驗之權衡軸,實作不得使其中任一路退化為不可行。

### 3.3 誤差特徵(憲章五:可分辨)

- **R-EXP-06**:三類失敗配置須呈現可分辨特徵——**摩擦型**(原木):讀值系統性偏低且各次形狀不一致;**計時型**(脈搏/高速水鐘):對稱抖動、隨傾角加劇;**離群型**(木球):每次一個游走位置的突刺。特徵由附錄 A 樣式實現;不得以無結構亂數取代。

## 4. 誤差組合模型(R-DATA)

- **R-DATA-01**:讀值 = 基準增量 B(傾角) + S(計時,傾角)×計時樣式[run] + 槽面樣式[run] + 球體樣式[run],逐段相加,下限夾制為 1。
  - B:緩 (12,36,60,84)/中 (14,42,70,98)/陡 (22,66,110,154)。
  - S 嚴重度表(乘數 輕=0.5/中=1/重=2.5):水鐘 緩輕・中中・陡重;脈搏 緩中・中重・陡重×2(即乘數 5);音格 全輕。
  - 各維度樣式為 3 組×4 段之固定偏移表,見附錄 A。
- **R-DATA-02**:所有 fixture 以機器可讀 JSON 置於 `/data`,結構見 §8;實作不得於執行期以亂數生成讀值。
- **R-DATA-03**:實作須內建「矩陣一致性測試」:對通過矩陣列舉之各配置自動執行判定,結果須與矩陣完全相符。

## 5. 規律指認與主張系統(R-JUD)——門檻 3 的機制化

原則:系統做記帳,玩家做看見(憲章二)。系統**永不**主動宣告「這是平方律/奇數律」。

- **R-JUD-01 選集**:玩家自筆記勾選 1–3 筆 run 組成「判定選集」。選集內配置不一致(球、槽面或傾角任一相異)時,系統顯示相異變因清單並拒絕受理判定,附語:「變因未受控,無法認證」(此為教學回饋,非懲罰)。
- **R-JUD-02 檢視工具**:選集可切換「累計/增量」視圖,並可將多筆 run 疊示於同一表格。工具僅呈現數字,不標示規律。
- **R-JUD-03 預測式指認**:玩家輸入「第五段增量之預測值」。以選集平均 Δ̄1 為基準,|預測 − 9·Δ̄1| ≤ 12%·9·Δ̄1 即指認成功;同時選集本身須通過內部一致性:Δ̄i/Δ̄1 與 (3,5,7) 之偏差各 ≤12%。兩者皆過 → 產生一筆「主張紀錄」(claim),標注其配置。
- **R-JUD-04 失敗回饋**:指認失敗僅回覆「預測未中/選集內部不一致」與偏差幅度,不解釋原因、不給正解(生產性失敗由玩家用比較工具自行診斷)。
- **R-JUD-05 斷言操作(E3 子項)**:
  - **E3.a 規律成立**:任一主張紀錄成立即點亮。
  - **E3.b 與球重無關**:玩家選取兩筆主張紀錄並執行「斷言」;系統驗證兩者**僅球相異**(銅大/銅小)且皆成立 → 點亮。驗證不符時列出實際相異變因。
  - **E3.c 隨傾角形式不變**(選做):同上,僅傾角相異 → 點亮。
- **R-JUD-06**:E3 於 evidenceState 之狀態 = {a,b,c 各自布林};「E3 確立」定義為 a∧b。

## 6. 論辯系統(R-DEB)——支柱 P3 單場

### 6.1 資料

辛普里奧支柱 P3 含三句證詞(佔位文即可,語意須保留):s1 重物性喜就下/s2 渴望愈強行愈疾,十斤之鐵思鄉十倍/s3 目的之因,數字豈能言之。弱點:**s2 × E3.b**。玩家證物欄固定含:E3(含子項顯示)、E1 存根、S1 存根(僅供錯誤出示之用)。

### 6.2 規則

- **R-DEB-01**:說服力量表初始 5,顯示於辯論廳。
- **R-DEB-02 追問**:對任一證詞免費追問,揭示補充佔位文;s2 之追問文須含「重量」語意線索。追問不改變量表。
- **R-DEB-03 出示**:出示 = 三元組(證據,子項,目標證詞)。E3 出示時必須展開選擇子項(a/b/c);存根證據無子項。
- **R-DEB-04 正解**:(E3, b, s2) 且 E3.b 已點亮 → s2 破裂 → 支柱 P3 破裂 → 勝利佔位畫面,場景結束。
- **R-DEB-05 方向對但不足**:(E3, a, s2) 且 b 未點亮 → 量表不減;證詞追加需求佔位文(語意:「單一球重之規律,何證與重量無關?」)——生產性失敗,指出缺口類型。
- **R-DEB-06 錯誤出示**:其餘任何三元組 → 量表 −1,附吸收佔位文。
- **R-DEB-07 中止**:量表歸零 → 辯論中止;顯示缺口提示(語意:「此支柱需要能將快慢與重量脫鉤的量化證據」);證據與主張狀態完整保留,可返回實驗台。
- **R-DEB-08 空手入場**:E3 未確立即可進入辯論;所有出示依上述規則自然導向 R-DEB-05/06/07——不設門禁,讓失敗自己教學。

## 7. 三層狀態機(R-STA)

- **R-STA-01** `evidenceState`:runs[](配置、讀值、序號、天數)、E3{a,b,c}、存根清單。
- **R-STA-02** `inferenceState`:claims[](選集、預測值、判定結果、配置指紋)、assertions[](類型、引用之 claim 對、驗證結果)、comparisons[](選取的 run 對與相異變因)。
- **R-STA-03** `beliefState`:simplicio.P3{s1,s2,s3 各 intact|broken;persuasion 0–5;status pending|broken|suspended}。
- **R-STA-04 關鍵轉移表**(事件→守衛→變更):

| 事件 | 守衛 | 狀態變更 |
|---|---|---|
| 執行實驗 | — | runs 追加;天數+1 |
| 判定 | 選集配置一致 | claims 追加(成敗皆記) |
| 斷言 b | 兩 claim 僅球相異且皆成立 | E3.b=true |
| 出示正解 | E3.b=true | s2=broken;P3=broken |
| 出示不足 | E3.a=true ∧ E3.b=false | 無量表變更;s2 追加需求文旗標 |
| 錯誤出示 | — | persuasion−1;=0 時 status=suspended |
| 再入辯論 | status=suspended | persuasion 重置 5;證詞狀態保留 |

- **R-STA-05**:切片內狀態存於記憶體即可;不要求持久化。所有轉移須為純函式可測(給定狀態+事件 → 唯一後繼狀態)。

## 9. 收尾與因果回述掛點(R-UI)

- **R-UI-01**:三面板皆可達且狀態即時同步;比較工具在筆記面板。
- **R-UI-02**:P3 破裂後顯示結束畫面,含兩題自由輸入(僅存於記憶體,不評分):「你改了什麼讓數據變好?」「這組證據還不能證明什麼?」——供形成性試玩之因果回述與證據邊界觀察使用。

## 10. 驗收準則對照

隱藏驗收測試將僅著錨於本文件 R-ID;凡未編 R-ID 之敘述皆屬說明性文字。評審五標準+門檻 3+結構檢驗之對應:一次能跑(R-ENV-01)、因果可讀(R-JUD-04、R-EXP-06、R-DEB-05/07)、維護成本(現場變更題,基於 R-DATA 資料驅動性)、違憲檢查(對照憲章)、門檻 3(R-JUD-03/05)、結構檢驗(R-EXP-05)。
