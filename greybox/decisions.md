## GB-ADR-001|資料載入策略:.js 全域賦值 + .json 規範鏡像

**背景**:R-ENV-01 要求雙擊 `index.html` 可玩;R-DATA-02 要求機器可讀 JSON 於 `/data`。file:// 之 opaque origin 使 fetch/XHR/ES module 載入一律被現代瀏覽器阻擋(Sol 審稿 #3)。
**選項**:①改用 localhost 指令啟動;②建置時 bundle;③資料以傳統 script 全域賦值載入,JSON 為規範鏡像。
**裁決**:採③。`/data/*.js` 為執行載體,`/data/*.json` 為規範鏡像,兩者深度相等性入自動測試(R-DATA-05)。
**理由**:①違反「雙擊可玩」對非技術試玩者的意義;②違反「無建置步驟」;③零依賴、零指令,漂移風險由測試封殺。

## GB-ADR-002|fixture 全面重校準與誤差語義修正

**背景**:v0.1 附錄 A 經枚舉驗算有 12+ 處矩陣違規(Claude 審核 A-1;Sol 審核 #2 同時指出)。根因:樣式為絕對偏移、容差為相對百分比——傾角越陡基準越大,相對衝擊縮水,一切變得可判定。另 v0.1「水鐘=對稱抖動」與矩陣「陡+水鐘三次平均不通過」邏輯矛盾:零均值抖動經平均必然收斂通過。
**裁決**:各樣式按「嚴重度×樣式÷基準」之相對衝擊重校;水鐘誤差改為「首段系統性偏高+微抖」(平均不可消除);脈搏改為「大幅抖動+輕偏移」(平均可大幅消除=廉價工具×多次路徑);原木後段流失遞增且三 run 形狀相異;木球突刺逐 run 游走。
**理由**:憲章四(不得製造錯誤的物理模型或虛假歷史因果)+R-EXP-04(矩陣為規範)。驗證:54 配置×216 判定點全符,與 12% 線最小距離 0.5pp,測試常駐 `/tests`。
**修訂(2026-07-19,依 Sol 第 2 輪審稿 A-1)**:撤除原裁決文中「此即伽利略需要緩斜面的史實原因」之宣稱。計時誤差樣式定位為**表現短時測量不確定性的教學代理**,非伽利略水鐘實測誤差之重建——本模型無操作程序與量測方程,無從導出該歷史因果;伽利略之實際程序(全程/指定距離集水稱重)與本切片逐段增量讀值不同。規格 v0.1.2 R-EXP-06 已同步代理聲明;史實頁揭露義務入 `docs/debt.md`。數值與矩陣不受影響。

## GB-ADR-009|試玩修正:診斷回饋、連續工作階段與辯論複盤(2026-07-20)

**背景**:總監實玩發現三個同源問題:①「選集內部不一致」會讓人誤以為多跑幾次即可救陡坡水鐘,但引擎實際判定的是平均形狀對奇數律的偏差;②A2-2/A2-3 三個 embed 被演成反覆進出實驗室;③辯論「追問」沒有可見產物,說服力歸零又把已持有全部證據的玩家送回實驗台做無效工作。
**裁決**:①判定回饋拆為「前四段形狀偏差」與「第五段預測偏差」兩道 12% 門檻;公開計時器性格,連續失敗第 2 次給中性觀察、第 3 次探索模式給診斷問句(學者模式按需索取),不替玩家宣判答案。②A2-2→A2-3 視為一次連續工作階段,僅首次進實驗台設確認閘;SC-R1 仍獨立進場。③追問改為可持久的 `pressed+insight`;介面採證詞卡、證據手牌與自然語言行動句。④**說服力歸零**=證據—證詞錯配,進 A3-F 與伽利略複盤後再入;**信譽歸零**=行為失格,仍進 SC-R1 做一筆乾淨紀錄。兩種資源不得共用同一修復動詞。
**不變邊界**:實驗 fixture、12% 判定線、證據取得條件、支柱弱點、說服力/信譽扣值與凍結劇情結論均不變;本次改動屬表現、回饋與失敗迴路修正。
**驗證**:資料鏡像、自動通關、追問狀態、失敗分流、雙歸零、UI 靜態契約與瀏覽器 G7 人工清單常駐。

**背景**:總監通關後判定辯論偏難——根因非物理,而是 UI 同時要求解三層謎(哪句可檢驗×八張牌×E3 子項),盲猜組合遠超 5 點說服力;章末回顧像線上問卷,收束感不足(Sol 彙整案,總監核准,Claude 實作)。

## GB-ADR-017|證據隨台詞入鏡＋角色語氣可讀性(2026-07-21)

**背景**:總監實玩第二章砲術圖場景時,角色正指著圖辯論,畫面卻只剩工作室背景;玩家被要求判讀自己看不見的證據。同輪又發現第二章把 ADR-005「老派書面語」做成半文言,連伽利略也受感染,使物理謎題多疊一層語文解碼。
**裁決**:①`lineFocusVisual` 為跨章資料契約:只在角色實際拿起、指向、比較圖像/器材時自動入鏡,同場景保留、下一個特寫取代、換場清除;選項與對話 z 層永遠高於特寫。第一章補 S1/S2/E1–E5 與器材關鍵時刻,第二章補 S3/S4/F2–F4 與證據牆。E2 綁縛悖論沿用程式 SVG,不另造可能畫錯物理資訊的點陣圖。②第二章 CH2-CR-002 僅調語氣、不動情節與判定:辛普里奧保留「老夫」與古派修辭,但不用「爾等／此乃／焉知」作理解門檻;伽利略、旅人與玩家選項採自然白話。唯一半文言例外=B2-3 伽利略刻意模仿,須有明確表演框架。
**理由**:遊戲若要求玩家讀圖,圖就必須在同一注意焦點內;謎題的難度應來自證據配對與物理推理,不應來自解碼文言。角色風格由節奏、立場、慣用詞與動作建立,不是靠提高閱讀門檻。
**驗證**:雙章關鍵場景/資產/替代文字/換場清除契約;E2 SVG 單一來源;第二章半文言禁詞掃描(刻意模仿節點唯一豁免);完整回歸常駐。

## GB-ADR-016|直向自動橫置(2026-07-21)

**背景**:玩家回報安卓「進不去」;總監截圖定位根因——直向時 16:9 舞台成 219px 高帶,標題卡上下溢出被裁,繼續/開始按鈕不可見;rotateHint 被關閉後即卡死。總監問「一進去可以直接是橫屏嗎」。
**裁決**:網頁無法命令裝置轉向(iOS 無鎖向 API;Android 需全螢幕手勢),採遊戲業標準解:**portrait+coarse 時整座 #stage 旋轉 90°(translate 置中+rotate)畫滿螢幕**,寬高公式=原式 vw/vh 對調(dvh 優先,vh 回退);實體轉向即由真橫屏 media 接手。rotateHint 於此模式退場(GB-ADR-014 的攔截職責被本案取代;原則 38 的層級要求仍適用於其餘 overlay)。已知代價:直向下鍵盤輸入(押注值/回顧/書信碼)為側向打字,可接受;字級 clamp 之 vw 縮小由 px 下限兜底。
**驗證**:契約測試(media 條件+rotate+rotateHint 退場);真機=總監 Android/iPhone 直拿實測,見標題全貌並可開局。
**補記(2026-07-21,總監授權 Claude 代修,Sol 事後複驗)**:系列首頁 v2 驗證揭露整合縫——media query 依視窗非旋轉舞台,直拿手機命中「窄高」分支跑出直版單欄。修法=**旋轉手機視同 844×390 低高度橫屏**:全部 13 個 `(max-height:520px)` 系列區塊追加替代式 `(orientation:portrait) and (pointer:coarse) and (max-width:520px)`(max-width 限定=視覺高度<520,iPad 直拿不誤吃手機壓縮版);全部 5 個窄高/窄寬區塊加 `(pointer:fine)` 鎖桌機。真橫屏手機規則零改動。契約測試鎖兩式。

## GB-ADR-027|證據圖片以穩定 ID 接線，並與實際台詞同步（2026-07-22）

**背景**：第一章實機重玩時，「介質阻力辨析」已取得卻未在舞台上出圖，旅人筆記六張證據也全退回空白模板。根因有二：表現層在台詞進入佇列時便搶先觸發特寫，換場或後續事件可能在玩家真正看到台詞前清除圖片；筆記本則從已翻譯的中文名稱反猜 `E4` 等資料 ID，名稱一改便無法找到 `card_E4`。

**裁決**：①章節 UI 以結構化 `{code,name}` 清單把證據交給舞台層；`code` 僅供資產與狀態解析，玩家畫面只顯示 `name`。②台詞事件攜帶原始 `scene`，證據特寫改在逐字機真正開始播放該句時觸發，不再於事件排隊時搶跑。③介質阻力辨析須等紙張與球體兩組對照都完成、正式取得 E4 時才出圖；不把半套實驗提前包裝成完整證據。

**驗證**：常駐測試禁止從顯示文字切割證據 ID，鎖定結構化接口與 `startLine` 特寫時序；第一章所有既有證據卡仍依 `card_<code>` 解析，缺圖才允許使用通用模板或 E2 SVG 降級。

**實機補記（2026-07-22）**：第三章取得 S5 時再次暴露 runtime 時序縫：證據事件雖已結構化，仍由 `setState` 當下直接送出，隨後 `renderAll → setScene` 會把剛出現的特寫清掉；因此旅人筆記有圖，取得當下卻看不到。修正為共用兩路：有取得台詞者先把 `{code,name,sceneId}` 附在該句事件上，直到逐字機真正開演才入鏡；實驗／辯論等沒有取得台詞者則延後到同一輪畫面完成後補發。三章 E／F／G／S 全走同一個狀態差分接口，禁止再由各章自行觸發。取得特寫加上「新證據已收入旅人筆記」標記，讓當下展示與永久收藏的關係可見。

## GB-ADR-028|三章章尾四拍接力（2026-07-22）

**背景**：前三章都有史實尾鏡，但功能不一致：第一章月球鎚羽沒有把「無空氣」與介質阻力說透；第二章月球高爾夫只收束拋體，沒有把「離手後仍向前」接到第三章；第三章凍結劇本已有月亮未解問題，runtime 卻漏接，章末卡又以「下一頁仍未寫定」等通用空話代替。畫面存在不等於敘事任務完成。

**裁決**：章尾固定四拍：①本章證據收束；②史實未來回聲，明說它驗證哪個條件與邊界；③從本章答案自然長出下一個未解問題；④提供明確操作出口。第一章補明月球無空氣使羽毛不再被介質拖慢，沿既有「向前又向下」進第二章；第二章保留阿波羅 14 月球高爾夫，新增「球桿離開後為何仍向前」並接第三章船桅；第三章補回「月亮為何沒有沿直線離開」，但不顯示尚未核准的第四章章名、日期或人物。

**出口規則**：下一章已可玩時，章末主按鈕直接進下一章，系列首頁為次要出口；下一章尚未完成時，只保留未解問題並回系列首頁。玩家可見文字禁止「鉤子」「未寫定」「旅程將繼續」等製作語或通用佔位句。

**驗證**：三章 runtime、凍結劇本變更紀錄、終幕卡與設計原則同步；常駐測試鎖定三個史實／未解問題語意錨、兩條章間路由、第三章不提前公布未核准章名。
